由于笔者的实验环境采用的是WSL2 Ubuntu 18.04版本，因此Makefile文件中的路径需要根据环境进行更改。

【参考代码完善】
该文件夹下存放的是对课设指导书中参考代码伪代码的具体复现，但该代码由于存在缓冲区溢出漏洞，因此只作为该漏洞的引出进行使用。
*编译* 直接在该文件夹下执行“make”指令即可；
*设备申请* 执行“sudo mknod /dev/BFDev c 369 0”进行设备文件申请；
*安装* 执行“sudo insmod BlockFIFODev.ko”进行驱动安装；
*测试* 执行“sudo ./test_overflow”后使用“sudo dmesg -c”查看即可发现溢出；
*卸载* 执行 “sudo rmmod BlockFIFODev”进行驱动卸载。

【修正后驱动代码】
对缓冲区溢出漏洞进行了修正。
编译安装与卸载与上述描述一致，不赘述。
测试文件集中在【TestFile】文件夹中，直接执行即可。具体测试样例及其作用和预期结果见报告中。
